What is in there?

Around 370'000 images are available for search and navigation. 330'000 comes from the automatic processing of the Cini Foundation photo-collection, so their metadata is in Italian and with possible typos coming from the Optical Character Recognition Process. The other 40'000 were imported from the Web Gallery of Art and have English metadata.

As such, it is heavily focused on Venetian art, and we advise you to search for things in Italian.

Interface

The interface is not the most self-explanatory unfortunately, so we recommend to go over this page quickly to familiarize yourself with the possibilities. It allows the combination of two visualisation modes and three(+) search techniques.

The top-left tabs are for defining the search query, the two others top blocks (current selection and negative selection) describe the images currently selected which will be used for the visual search queries. Below is the area where the search results are displayed.

Visualisation modes

List view

The default result view, where images are displayed on a list with their author and description fields. Clicking on the image of the artwork gives the full metadata information available. Clicking on the logo of the provenance takes you back to the original document or the webpage of the corresponding item.

For each object, the "+" button adds the image to the current selection, the "-" button to the negative selection, and the "x" button remove the image to any selection it is part of. (See "Search modes"->"Image Search" for the usage).

Embedding view

The embedding view is another way of representing the current results (from any search) as a visualization map. The images are organized in a visually consistent manner, which can gives a global view of a class of images. The image above for instance represents the 550 images coming from the textual query "Bellini Giovanni Madonna". It can be activated/deactivated with the Show embedding toggle-button on the left.

It is possible to zoom and navigate the view with standard google-map-like controls. Left-clicking on an image adds/removes the image to the current selection while right-clicking adds/removes the image to the negative selection.

Trick: only a limited number of results are shown at a given time which might be smaller than the total number of results. By default, only 250 images are shown at a time, which might not be enough but allows your computer not to lag too much. If you want to show more, the Options tab allows you to increase this number. However, do not expect to visualize more than 700-800 elements at the same time with the current implementation.

Some bugs are still present, if you see some weird visual artifacts, just deactivate then reactivate the embedding view, it usually solves display issues.

Search modes

The top-left tabs show the different search capabilities, it relies on three different search modes, a text metadata search, a global image search, and an experimental detail search. Each of them has a separate tab on the top left.

Textual search

This is a standard textual search on the author and description fields of the objects indexed in the database. It uses some fuzzy matching to also return approximate version of titles and authors, so that typos in the data are not preventing us from finding the images. An experimental filter on a date range is also possible, but it usually relies on the identified name of the artist, and uses the corresponding birth and death date.

What is important is that the textual search parameters allow to pre-filter the visual search if the filter with metadata box is checked on the two other search modes. So for instance, you can search for similar images in the production of a given artist (only the images of 'Giovanni Bellini' for instance) or for a given scene (only the 'Madonna col Bambino'). This can help a lot with the quality of the results by drastically reducing the number of candidates to compare the visual query to, compared to going through the total 370'000 elements.

Image search

The image search is only available if at least one image was selected as part of the current selection. The simplest form of search is when you just have one image selected. In that case, the search system tries to automatically find a matching area between the query image and the target images.

You can also search with multiple images at the same time, both by using images which should be similar using the images in current selection, and dissimilar images with the images in negative selection.

We actually recommend to use the search with a single element in the case of trying to find a pattern reappearing, a close variation or copie. The multiple images search is better suited for looking for sligthly more abstract concept in an iterative way: start with a couple examples and try to find other versions of the same concept. It also synergizes well with the embedding view, as adding positive and negative examples is easy.

Detail search

For the detail search, you need to have only one element in the current selection. You can then click on the Select Image Region button to choose the area of the image you want to use as a query. Unfortunately, the system is not able to track elements which are toot small, so a region which is 20% of the biggest size of the image is roughly the smallest you can expect unfortunately.

Groups

When you have found a set of images you want to keep for further study, you can save them to a group with the Save to Group button. Every image in your current selection will be added to a newly-created (or an already existing) group. You can then inspect them further on the My Groups link on the top right of the interface.

For each group, you can either just get a list view of the elements, where you can delete elements, or add notes to the group. You can also have an embedding view, where an initial positionning is computed for you, then you can drag images around according to the point you want to put across from this set of images. Try it out!

Some examples

Simple search example: type "Bellini madonna" as a text search, select the first image in your selection (or simply click here to get the image directly). Then use the Image Search tab to search with this single image. You should get all the copies/variations of different artists. You should see the results below.

More complex: start with a standard Bassano work (for instance this one). Use "Bassano" as a search query to only search through the Bassano works, go on the Detail Search tab, check the filter with metadata toggle (so that the system will only search through the Bassano works), and select a box around the kneeling lady in the front before pressing search, you should see the results below.

Duplicate Comparisons

Sometimes it is hard to detect if two photographs are from the same artwork or not, given the ability of certain painters to replicate their own work. For this, you can select two images in your Current Selection, and use the well-hidden button in "Options"->"Compare 2 Images". For instance, you can visualize the tiny differences between two famous Brueghel works here.

NB: this a slow process and only works for very similar images, close copies or before/after restoration.
